Since ACME product generation times can vary significantly (sub-second to > 2 minutes), the more common transaction sequence is a bit more involved.

  1. POST a request to ACME service, capture JSON Order Number response.
  2. Iteratively GET the the progress of the product generation by referencing the Order Number, capture the JSON progress information. This can be used to feed into a progress bar feedback window for the client.

When the progress is > 5%

  1. GET the first frame (or any frame, with reasonable patience) by referencing the Order Number

When the progress is = 100%

  1. GET the final prodcut

For example, a requesting service could:

POST: http://service.acme.codes/new?msg=GreetingsCustomer!

ACME service would return JSON:

{orderNumber: 3284729}

Now the requesting servcice can retrieve the progress of the order:

GET: http://service.acme.codes/orders/3284729/progress

ACME service would return JSON:

{progress: 12}

When reported progress from the above call is 100, the requesting servcice can retrieve the final product:

GET: http://service.acme.codes/orders/3284729/gif
